# wez-html 操作手冊

> 給**內部同事自助操作**用的一步步指南。照做就會，不用懂技術。
> 服務網址：`http://10.1.1.7:8090/`（要連公司 VPN 才開得起來）

![wez-html 首頁](images/dashboard-v1.2.0.png)

---

## 這是什麼？我為什麼要用它

一句話：**把一個 `.html` 檔丟上去，馬上拿到一個公司內部人人打得開的網址**，永久常駐、不會自己消失。

以前要給同事看一個 demo、一份互動報表、一頁問卷，得找工程師幫忙架站。現在你自己拖一個檔就好。

**適合放這裡的東西**

- demo / 提案 / 個人賽作品分享
- 一頁式的互動報表、投影片、問卷、記分板
- 想給同事一個「打得開的連結」而不是寄一個檔案

**不適合放這裡的東西**

- 對外公開的正式官網（這只在公司內網，外面連不到）
- 需要帳號登入、存重要資料的系統（沒有備份、沒有權限控管）
- 機密資料（內網信任環境，同事之間看得到）

## 開始前：先確認兩件事

### 1. 連上公司 VPN

打開瀏覽器，網址列輸入 `http://10.1.1.7:8090/`。

- **開得起來** → VPN 正常，可以繼續。
- **開不起來 / 一直轉圈** → 你沒連到公司 VPN。連上 VPN 再試一次；還是不行找 IT。

### 2. 想好你的「識別名」(identity)

上傳時要填「你是誰」，用你的**英文名或工號**（例如 `vincent`、`yc`、`e12345`）。

它的用途只有一個：**只有當初上傳的人，能刪掉自己的站台**。所以：

- 用你自己固定會記得的名字，別每次亂填。
- 不要抄範例裡的 `yc`，那是別人的。

---

## 最簡單的做法：拖檔上傳（推薦，非工程同事看這段就夠）

全程在瀏覽器裡點一點，不碰任何指令。

### 步驟

1. 連上 VPN，瀏覽器打開 `http://10.1.1.7:8090/`。
2. 找到中間那塊**紅色虛線框**（寫著「拖 `.html` 單檔到這裡」）。
3. 把你的 `.html` 檔**直接拖進那個框**；或點框裡的「或點此選擇檔案」去挑檔。
4. 填下面三格：

   | 欄位 | 填什麼 | 範例 |
   |---|---|---|
   | **專案名稱** | 給人看的中文名字，方便自己在列表裡認 | `六月客戶提案` |
   | **Site name** | 網址會用到的英數短名（只能 `a–z 0–9 -`） | `june-proposal` |
   | **你是誰** | 你的英文名 / 工號（見上面「識別名」） | `vincent` |

5. 按紅色的 **「上傳」** 按鈕。
6. 上傳成功後，你的網址就是 `http://10.1.1.7:8090/<你的 Site name>/`，例如 `http://10.1.1.7:8090/june-proposal/`。把這條連結貼給同事就行。

> **只能拖單一個 `.html` 檔。** 如果你的網站是「一個資料夾、裡面有多個檔案 + 圖片」，拖檔上傳不吃，要走下面的〈進階：整個資料夾上傳〉。

### 撞名了怎麼辦

如果你的 Site name 已經被別人用過，系統會擋下來。兩個做法：

- **換一個 Site name**（最簡單，例如加個後綴 `june-proposal-2`）。
- 如果那個站**是你自己之前傳的**、想覆蓋更新 → 把左下角的 **「強制覆蓋(撞名時)」** 開關打開，再按上傳。

> ⚠️ 「強制覆蓋」會把那個 Site name 底下**原本的東西整包換掉**。確定是你自己的站、要更新，才開這個。

## 看看我傳過哪些站 / 刪掉不要的站

### 看列表

首頁往下滑，**「目前的站台」** 區塊就是所有人傳過的站，看得到：專案名稱、Site、是誰傳的(uploader)、上傳時間、大小。

### 刪站

在列表裡找到你的站，點右邊的**刪除**按鈕。

- 系統會要你確認你的**識別名**（就是當初上傳填的「你是誰」）。
- 識別名對得上 → 刪掉。**對不上 → 刪不了**（你只能刪自己的站，不能動別人的）。

> 站台上傳後**永久常駐、不會自動下架**。不要的自己刪，不然會一直佔著。

## 進階：用 Claude Code plugin 一句話生網站

如果你有裝 [wezoomtek Claude Code plugin](https://github.com/yanchen184/wezoomtek-claude-code-plugin)，可以在 Claude Code 裡直接打，連 HTML 都不用自己寫：

```
/wez:upload-html ai-report.html yc          ← 推一個已經有的 .html
/wez:upload-html yc "報名表，收姓名 email 部門，後台看筆數"   ← 一句中文，Claude 幫你寫好整頁網站再推上去
/wez:upload-html --list                     ← 列出你傳過的站
```

第二種（一句中文需求）最省事：Claude 會把整頁 HTML、含資料儲存功能都寫好再推，你只要描述要什麼。

## 進階：整個資料夾上傳（多檔 / 含圖片）

拖檔只能單一 `.html`。如果你的網站是一整個資料夾（多個頁面、圖片、`screenshots/` 之類），要用命令列工具 `wez_upload_html`。

> 這個工具要先跟 plugin 維護者拿對應你系統（Mac / Windows / Linux）的檔案，放進 PATH。沒有它就只能傳單檔。

```bash
# 把整個資料夾推上去
wez_upload_html ./我的網站資料夾 你的識別名 --name 站台短名 --server http://10.1.1.7:8090

# 撞名要覆蓋（確定是自己的站才加 --force）
wez_upload_html ./我的網站資料夾 你的識別名 --name 站台短名 --force --server http://10.1.1.7:8090

# 列出所有站
wez_upload_html --list --server http://10.1.1.7:8090

# 刪站（識別名要對得上）
wez_upload_html --delete 站台短名 你的識別名 --server http://10.1.1.7:8090
```

> `--server http://10.1.1.7:8090` 每次都要帶，不然它會去找你自己電腦而不是公司這台。

## 加分功能：每個站附一個小型資料庫（KV）

每個站台都自帶一份輕量的「鍵-值」儲存（KV），可以讓你的網頁存一點 demo 等級的資料 —— 記分板分數、留言、投票結果、簡單設定。

用 Claude Code plugin 的「一句中文需求」生網站時，這些會**自動接好**，你不用管技術細節。

如果你自己寫網頁要用，API 是（`<site>` 換成你的 Site name）：

```
GET    /<site>/api/kv           列出所有 key
GET    /<site>/api/kv/<key>     讀一個 key
PUT    /<site>/api/kv/<key>     寫一個 key（body 要是合法 JSON）
DELETE /<site>/api/kv/<key>     刪一個 key
```

**限制**（demo 夠用，別當正式資料庫）：

- 單一值 ≤ 256 KB
- 一個站最多 1,000 個 key、總共 10 MB
- **沒有帳號驗證** —— 同站台的人都讀寫得到，別放機密
- 刪站台時 KV 資料一起刪

## 疑難排解

| 你看到的訊息 / 症狀 | 多半是什麼原因 | 怎麼解 |
|---|---|---|
| 網址 `http://10.1.1.7:8090/` 打不開、一直轉圈 | 沒連公司 VPN | 連上 VPN 再試；還是不行找 IT |
| 上傳按了沒反應 / 逾時 | VPN 沒通，或服務暫時掛了 | 先確認瀏覽器開得開首頁；開不開就是 VPN / 服務問題 |
| 提示「site 已被 XXX 上傳」（撞名） | 你的 Site name 被別人用過了 | 換一個 Site name；若那站是你自己的、要更新 → 開「強制覆蓋」 |
| 刪不掉、說識別名不對 | 你填的識別名跟當初上傳的不一樣 | 只能刪自己的站；不確定是誰傳的，看列表的 uploader 欄，或問對方 |
| 拖檔說只支援單檔 | 你拖的是資料夾或多個檔 | 資料夾要走〈進階：整個資料夾上傳〉的命令列工具 |
| 網頁上傳後圖片破圖 / 少東西 | 你的網站其實是多檔（HTML 引用了外部圖），但只傳了單一 HTML | 改用資料夾上傳，把圖片一起帶上去 |
| 網頁載入後樣式跑掉 | 網頁引用了外部 CDN（如 Google Fonts、線上圖表庫），內網連不到外網 | 網頁要做成「不依賴外部網路」的自包含單檔；請寫網頁的人 inline 所有資源 |

以上都試過還是卡 → 找 **plugin / 服務維護者**。

## 幾條要記住的規矩

- **要連 VPN** 才用得了（`http://10.1.1.7:8090/`）。
- **識別名固定用自己的**，只有你能刪自己的站。
- **站台永久常駐、不自動下架** —— 不要的自己刪。
- **內網信任環境，別放機密**：識別名只是追溯用不是密碼，別人知道你的識別名就能覆蓋 / 刪你的站。
- **網頁要自包含**：不要引用外部 CDN，內網連不到外網會破版。
- **不是正式主機**：沒備份、沒 SLA，只拿來 demo / 內部分享。
